решается таким путем:
1. Кто первый занял, того и тапки
2. name занимается на срок до 4-х недель
3. начиная с третьей недели начинается челендж, кто займет имя
4. по окончанию 4-х недель, имя отдается тому, кто выполнил челендж

имя публикуется в DHT в namespace: name
в этот namespace можно попасть двумя путями:
1. занять пустое имя
2. выиграть челендж

Челендж начинается путем публикации в name_challenge записи по правилам:
1. Участник вычисляет хеш blake3 от (name, public_key, nonce), подбирая такой nonce, чтобы удовлетворял правилу, что чем больше \0 в начале, тот и заменяет запись в name_challenge на свою (в случае совпадения difficulty, приоритет отдается текущему владельцу имени в name)

Когда челендж закончиться, запись из name_challenge переезжает в name

Если при истечении name не было записей в name_challenge, то запись освобождается

Может быть ситуация, что владелец имеет nonce с большей сложностью, но не участвует в challenge, тогда его имя замениться тем, кто занял запись в name_challenge.

Те, для тех, кто владеет именем давно должно быть легче "защитить" имя, либо, если человек очень хочет, то должен потратиться на то, чтобы выиграть это имя.
